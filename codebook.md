# Codebook: Final Analysis Panel

This codebook documents the main reproducibility dataset used in the thesis **"Market-Wide Liquidity Stress and Return Predictability in Cryptocurrency Markets"**.

The main dataset is:

```text
processed/analysis_panel.csv
```

It is generated by `notebooks/05_build_analysis_panel.ipynb` from the frozen `raw/` and `processed/` data bundle described in the README. The file is a token-day panel: one row corresponds to one CoinMarketCap asset (`cmc_id`) on one calendar day (`date`).

## Dataset Scope

The thesis combines three information layers:

1. **CoinMarketCap daily market data**: daily OHLCV prices, volume, and market capitalisation.
2. **CoinMarketCap metadata**: token category, platform, listing date, and tags.
3. **TORD offering characteristics**: static issuance-stage variables from the Token Offerings Research Database, harmonised to one first offering per `cmc_id`.

The analysis uses daily data rather than high-frequency order-book data because daily OHLCV information is available across a broader and more heterogeneous cryptocurrency universe.

## Sample Construction in the Thesis

The paper distinguishes between the broad construction panel and the final empirical sample used in the main results.

| Stage | Restriction applied | Assets | Token-days |
| --- | --- | ---: | ---: |
| 1 | Combined daily market panel for the TORD universe with usable price history | 1,853 | 1,076,915 |
| 2 | Valid first-offering record with start date | 531 | 319,174 |
| 3 | Activity filters: entry gate plus rolling activity filter | 420 | 231,354 |
| 4 | Coin/token category available | 295 | 192,570 |

The final working sample in the paper contains **192,570 token-day observations for 295 assets** from **2 April 2024 to 30 March 2026**. It consists of **83 coins** and **212 tokens**.

Rows excluded by the activity filters are still useful during construction because they initialise cumulative activity counts, rolling liquidity measures, and lagged variables. They are not used directly in the final estimation sample.

## How to Use the Panel

For the paper's main empirical sample, use:

```text
in_sample == 1
category in {"coin", "token"}
```

Additional complete-case filters are applied inside `07_analysis.ipynb` depending on the table, model, or figure. For example, stress-prediction models require the target and lagged feature set to be non-missing.

The most important target labels are:

- `stress_amihud`: baseline Amihud-only liquidity-stress label.
- `stress_amihud_strict`: stricter joint stress label combining high Amihud illiquidity and low turnover.
- `crash`: bottom-decile negative-return event relative to each asset's own expanding return history.

## Important Methodological Notes

- TORD variables are offering-stage characteristics and are repeated across daily market observations for the same token.
- Market observations before the recorded offering start are removed to avoid linking offering information to pre-offering trading days.
- Liquidity proxies are low-frequency daily proxies, not direct order-book measures.
- Rolling and expanding-window calculations are constructed with lagged information to reduce look-ahead bias.
- CoinMarketCap latest-snapshot variables are not used as forecasting predictors because they describe conditions at the final collection date.
- The predictive analysis derives additional lagged variables such as `ps_gamma_lag1`, `pc1_lag1`, `vol20_lag1`, and `spread_lag1` from the variables listed below.

### 6. Smoothed Liquidity and Volatility Measures

The paper uses rolling or smoothed liquidity proxies because daily raw values are noisy. Liquidity variables are winsorised at the 1st and 99th percentiles using in-sample observations.

| Variable | Source | Level | Description | Unit / Coding | Paper use |
| --- | --- | --- | --- | --- | --- |
| `amihud_med30` | Engineered | Token-day | Trailing 30-day rolling median of `amihud_raw`. | Ratio | Main Amihud illiquidity measure. |
| `turnover_med30` | Engineered | Token-day | Trailing 30-day rolling median of `turnover_raw`. | Ratio | Main turnover measure. |
| `spread_med30` | Engineered | Token-day | Trailing 30-day rolling median of `spread_raw`. | Ratio | Fallback spread measure. |
| `cs_spread_30d` | Engineered | Token-day | Trailing 30-day rolling median of `cs_spread_raw`. | Ratio | Main Corwin-Schultz spread proxy. |
| `amivest_30d` | Engineered | Token-day | Trailing 30-day rolling median of `amivest_raw`. | USD per return unit | Descriptive market-wide factor input. |
| `volatility_20d` | Engineered | Token-day | Trailing 20-day rolling standard deviation of daily returns. | Decimal return volatility | Descriptive statistics, crash models, prediction features. |
| `ps_gamma` | Engineered | Token-day | Pastor-Stambaugh-style rolling reversal coefficient using signed flow. | Coefficient | Lagged predictive feature. More negative values imply stronger reversal effects. |

## Variables Not in the Final Analysis Panel

The upstream raw and processed files contain additional CMC metadata and TORD fields such as token names, symbols, CMC latest-snapshot price/supply/rank variables, smart-contract addresses, source labels, country, raw URLs, and detailed minimum-investment parsing fields. These are not documented as final `analysis_panel.csv` variables because they are not retained in the panel produced by Notebook 05.

This distinction matters because the thesis uses CMC latest-snapshot variables only as part of the data-construction record, not as predictors for historical forecasting. Using point-in-time snapshot variables as predictors for earlier dates would introduce look-ahead bias.

## Caution Notes

The following variables should be interpreted carefully because of source sparsity, skewness, or noisy reporting:

- `raised_usd`
- `price_usd`
- `distributed_in_ico_frac`
- `ico_duration_days`
- `rating`
- `min_inv_usd_value`
- `market_cap` and `log_mcap` when CMC market capitalisation is unreported
- `turnover_raw` and `turnover_med30`, which depend on reported market capitalisation
- `ps_gamma`, which requires enough rolling observations for stable estimation

## Summary

The final thesis codebook describes a daily token-day analysis panel linking CMC market data, selected CMC metadata, and harmonised TORD first-offering characteristics. The paper's final working sample contains 192,570 token-days for 295 assets from 2 April 2024 to 30 March 2026. The main empirical variables measure daily returns, size, low-frequency liquidity proxies, asset-specific liquidity stress, crash events, and market-wide liquidity conditions.
